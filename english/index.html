Mitech's data is permanently stored in blocks
The blocks are generated and linked successively in time sequence, and each block records all transaction information that occurs during the creation period. Block data structure is generally divided into district size (header) and block (body), Mitech district's link to the previous blocks and to ensure the integrity of historical data through the time stamp characteristics; block contains all transaction information generated after verification of the block, in the process of creating.
Our core technology
Communication: Mitech uses P2P technology to organize all network nodes. Each node realizes functions such as routing, new node identification and data dissemination through multicast, and intelligently identifies data information and completes sensor needs.
Storage: Mitech data is stored in memory in a block chain data structure during runtime and will eventually be persisted to the database. For larger files, it can also be stored in a file system outside the chain, and the digest (digital fingerprint) is saved to the chain for self proof.
Security mechanism: Mitech system uses a variety of cryptographic principles to encrypt data and protect privacy. Completely surpassing other block chain systems involved in financial applications, high and reliable security algorithms reach the national level, leading the world in efficiency.
Consensus mechanism: each node in the Mitech system achieves a consistent strategy and method, which is selected flexibly according to the different types of system and the application scenario. PBFT the consensus mechanism allows Byzantine fault tolerance. The consensus mechanism allows the participation of strong regulatory nodes, has the ability to classify the authority, the performance is higher, and the energy consumption is lower.
